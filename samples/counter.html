bp-ui first component

Counter UI built once, then reused in vanilla JS and Alpine.

This page defines the `bp-ui` contract for future components: shared `bp-ui-*` classes, one framework-agnostic behavior core, library-owned CSS, and ready-made snippets that can be pasted into future projects.

One DOM contract Tailwind optional Alpine adapter over shared core

Default imports

import '@braudypedrosa/bp-ui-components/styles';
import { initBPUICounters } from '@braudypedrosa/bp-ui-components';

Alpine usage loads the same package before Alpine starts so window.bpUICounter(...) is available to x-data.

Future component rules

The counter is the reference pattern for the rest of the library.

  • Structure: `core` for behavior, `components` for vanilla DOM, `alpine` for adapters, `styles` for library CSS, `samples` for copy-paste demos.
  • Naming: `.bp-ui-{component}`, `.bp-ui-{component}__{element}`, `.bp-ui-{component}--{modifier}`, `.is-*`, and `data-bp-ui-{component}`.
  • Snippet-first: publish ready-made HTML first, then keep the JS APIs stable behind it.
  • Tailwind optional: wrappers can use utilities, but the component internals cannot depend on them.
  • Single behavior source: Alpine must adapt the same state rules as vanilla, not fork them.

Canonical vanilla snippet

This is the stable snippet the library auto-initializes.

<div class="bp-ui-counter" data-bp-ui-counter data-value="2" data-min="0" data-max="8" data-step="1">
  <div class="bp-ui-counter__label">Seats</div>
  <button class="bp-ui-counter__button bp-ui-counter__button--decrement" type="button" aria-label="Decrease seat count">-</button>
  <div class="bp-ui-counter__field">
    <input class="bp-ui-counter__input" type="number" inputmode="numeric" aria-label="Seat count" />
  </div>
  <button class="bp-ui-counter__button bp-ui-counter__button--increment" type="button" aria-label="Increase seat count">+</button>
  <div class="bp-ui-counter__hint">Shared markup, no framework required.</div>
  <div class="bp-ui-counter__status" hidden></div>
</div>

Canonical Alpine snippet

Same `bp-ui-counter` internals, with Alpine state and Tailwind around the outside.

<script type="module">
  import '@braudypedrosa/bp-ui-components/styles';
  import '@braudypedrosa/bp-ui-components';
</script>
<script defer src="https://cdn.jsdelivr.net/npm/alpinejs@3.x.x/dist/cdn.min.js"></script>

<div class="rounded-[1.5rem] border border-slate-200 bg-white p-5 shadow-sm">
  <div
    class="bp-ui-counter bp-ui-counter--soft bp-ui-counter--md bp-ui-counter--horizontal"
    x-data="window.bpUICounter({ value: 3, min: 1, max: 12, step: 3, variant: 'soft' })"
    x-init="init()"
    role="group"
    aria-label="Guest counter"
    :class="{ 'is-at-min': !canDecrement(), 'is-at-max': !canIncrement(), 'is-disabled': disabled, 'is-loading': loading }"
  >
    <div class="bp-ui-counter__label">Guests</div>
    <button class="bp-ui-counter__button bp-ui-counter__button--decrement" type="button" aria-label="Decrease guest count" @click="decrement()" :disabled="!canDecrement()">-</button>
    <div class="bp-ui-counter__field">
      <input
        class="bp-ui-counter__input"
        type="number"
        inputmode="numeric"
        aria-label="Guest count"
        x-model="inputValue"
        :min="min"
        :max="max"
        :step="step"
        @input="onInput($event)"
        @blur="commitInput($event)"
        @keydown.enter.prevent="commitInput($event)"
        @keydown.arrow-up.prevent="increment()"
        @keydown.arrow-down.prevent="decrement()"
      />
    </div>
    <button class="bp-ui-counter__button bp-ui-counter__button--increment" type="button" aria-label="Increase guest count" @click="increment()" :disabled="!canIncrement()">+</button>
    <div class="bp-ui-counter__hint">Tailwind can wrap the layout without owning the component styling.</div>
    <div class="bp-ui-counter__status" hidden></div>
  </div>
</div>
